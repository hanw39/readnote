## 一、Claude Design by Anthropic：AI 设计工具【⭐⭐】
**来源：** Anthropic News
**链接：** https://www.anthropic.com/news/claude-design-anthropic-labs
**关键词：** Claude Design、设计工具、视觉、原型、Anthropic Labs

### 摘要

Anthropic 推出 Claude Design，这是 Anthropic Labs 旗下第一款产品，基于 Claude Opus 4.7 视觉模型驱动，可以让用户通过对话协作创建设计、原型、幻灯片、单页等视觉作品。

**解决什么问题。** 有经验的设计师需要控制探索的节奏——很少有时间用十几个方向做原型，所以往往只探索几个。对于没有设计背景的创始人、产品经理、营销人员，自己做视觉内容就更困难了。Claude Design 让设计师有更大探索空间，也让其他人能做出视觉作品。

**核心流程。** 描述需求 → Claude 生成第一版 → 通过对话、内联评论、直接编辑或 Claude 生成的调节滑块来精炼。当获得代码库访问权限时，Claude 还能自动应用团队的设计系统。

**具体使用场景。**
- 真实感原型：把静态模型转换为可分享的交互原型，用于用户测试，无需代码审查或 PR
- 产品线框图和模型：产品经理可以画出功能流程图，交给 Claude Code 实现，或交给设计师进一步精炼
- 设计探索：设计师可以快速创建多个方向来探索
- 融资 deck 和演示：创始人/客户经理可以从大纲到完整 deck，导出为 PPTX 或发送到 Canva
- 营销物料：落地页、社交媒体素材等
- 前沿设计：可以用语音、视频、shader、3D、AI 构建代码驱动的原型

**设计系统集成。** onboarding 时 Claude 会读取代码库和设计文件，为团队构建设计系统，之后每个项目自动使用团队的颜色、字体和组件。

**与 Claude Code 的衔接。** 设计完成后，Claude 会打包成一个 handoff bundle，一键传给 Claude Code 实现。

**可及性。** 对 Claude Pro、Max、Team 和 Enterprise 订阅者开放研究预览，逐步向用户推送。

## 二、A2UI v0.9：Google 发布跨框架生成式 UI 标准【⭐⭐】
**来源：** Google AI (Developers Blog)
**链接：** https://developers.googleblog.com/a2ui-v0-9-generative-ui/
**关键词：** A2UI、生成式 UI、框架无关、Google、Agent SDK

### 摘要

Google 发布 A2UI v0.9，这是一个框架无关的生成式 UI 标准，旨在让 AI Agent 能够使用企业现有设计系统实时生成定制化 UI 组件，实现跨平台（Web、移动端）的低延迟流式生成。

**核心思路。** 从"标准组件"→"基础组件"：前端开发者不想用新的组件，他们有自己的设计系统。Agent 应该用现有的前端来响应，而不是生成新组件。A2UI 把这个理念明确为"Basic"组件集。

**主要更新。**
- **共享 Web-Core 库：** 简化了任何浏览器 UI 渲染器的实现；官方 React 渲染器已发布，Flutter、Lit、Angular 渲染器也更新了版本
- **Agent SDK for Python：** `pip install a2ui-agent-sdk` 即可集成，5 步完成 Hello World
- **新语言特性：** 客户端定义函数（验证用）、客户端到服务器数据同步（支持与 Agent 协作编辑）、改进的错误处理、简化的模块化 schema
- **传输层简化：** 支持 MCP、WebSocket、REST、AG UI、A2A 或任意自定义传输

**生态系统集成。** 已与 AG2 和 Vercel 集成，可以更方便地接入 Agent 生态。

**对开发者的意义。** A2UI 把生成式 UI 从 demo 阶段推向生产阶段，核心价值是解耦了 UI 意图声明和具体平台渲染——同一套 Agent 逻辑可以在 React、Flutter、Angular 等任意框架上渲染，企业可以用自己的设计系统，而不是被框架绑架。
