## 一、Windsurf 2.0：Agent Command Center + Devin 集成

**来源：** Windsurf Blog  
**链接：** https://windsurf.com/blog/windsurf-2-0

### 摘要

Windsurf（Codeium 被 Cognition 收购后改名）发布 2.0 版本，最大更新是推出了 **Agent Command Center**（Agent 指挥中心）并正式将 **Devin**（Cognition 自家的云端自主 AI 软件工程师）深度集成到编辑器里。

**Agent Command Center 是什么：**

一个全新的面板，以 **Kanban 视图**展示你运行的所有 agent（本地+云端），按状态分组。随着 AI agent 越来越强大，工程师的角色从"自己写代码"转变为"管理一队 agent"。这个界面就是为这个转变设计的——让你看到每个 agent 在干什么、什么被阻塞了、什么可以审查了。

**Spaces（空间）：**

新的工作组织单元。一个 Space 把某个任务/项目的所有相关内容聚合在一起：agent 会话、PR、文件、上下文。比如一个"登录流程重构"的 Space，可能包含一个本地 session 原型化 UI、两个云端 session 分别处理 API 改动和测试用例。Space 之间互不干扰，切换 Space 就是切换任务——只不过每个任务里都有一支 agent 团队在工作。

**Devin 集成：**

Devin 是 Cognition 开发的云端自主软件工程 agent，能独立完成 debug、部署、测试等工作，而且有自己独立的 VM、桌面和浏览器，关了笔记本它继续跑。

现在 Devin 直接集成进 Windsurf：你在本地和 agent 讨论好方案，一键交给 Devin 执行；Devin 开好 PR 后你可以在 Windsurf 里直接 reviewdiff、跑测试或让本地 agent 做收尾工作。

**核心理念：** 把完整的软件工程周期——规划、委托、审查——全部集中在一个工具里。

Devin 包含在所有 Windsurf 计划中（无需额外付费）。

## 二、Google Gemini CLI 新增 Subagents 支持

**来源：** Google AI  
**链接：** https://developers.googleblog.com/subagents-have-arrived-in-gemini-cli/

### 摘要

Google 宣布 Gemini CLI（Gemini 的命令行工具）新增 **subagents（子代理）** 支持。

Subagent 功能允许在一个主 agent 下创建多个子 agent，并行处理独立任务，每个子 agent 拥有自己的上下文。这对于需要同时处理多个方向的大任务特别有用——比如一个任务涉及 API 设计、前端实现、测试编写，以前可能需要手动分配，现在可以让主 agent 协调多个 subagent 同时工作。

Gemini CLI 是 Google 在 AI coding 工具领域的重要产品，这次加入 subagent 支持，使其在 agentic coding 能力上更接近 Claude Code 和 Windsurf。

## 📊 综合分析

今天的 AI coding 产品领域，两大玩家（Windsurf vs Google Gemini）都在做同一件事：**让人类工程师从"单独操作 agent"进化到"管理多个并行的 agent"**。

Windsurf 2.0 的 Agent Command Center 是这个方向上最激进的产品设计。它不只是加了一个功能，而是重新设计了整个交互模型——Kanban 视图、Spaces 概念、把 Devin（云端 agent）和本地 agent 在同一界面统一管理。这是第一次有一个 IDE 把"多 agent 协调"做成产品级功能，而不是小众极客玩法。

Gemini CLI 加 subagent 则代表 Google 迎头赶上的意图。Claude Code 之前已经展示了 subagent 的用法（4月10日的文章里详细写过），Gemini CLI 现在也有了类似能力。

两条放一起，最核心的趋势是：**2026 年的 AI coding 竞争焦点，已经从"单 agent 能力有多强"转移到了"多 agent 协作体验有多好"**。谁是最后的赢家，可能不取决于哪个模型最强，而取决于谁先做出最好的多 agent 协调体验。
