## 二、Claude Code v2.1.113 发版【⭐】
**来源：** Claude Code Changelog
**链接：** https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#21113
**关键词：** Claude Code、Changelog、发版

### 摘要

Claude Code v2.1.113 是一个重要的架构变更版本，主要变化：
- CLI 改为通过原生平台二进制文件（per-platform optional dependency）而非打包的 JavaScript 来启动 Claude Code
- 新增 `sandbox.network.deniedDomains` 配置，可以阻止特定域名（即使更宽泛的 `allowedDomains` 通配符本来允许）
- 全屏模式：Shift+↑/↓ 在延伸到可见边缘之外时现在会滚动视口
- 多平台改进：macOS 安全规则增强（`/private/{etc,var,tmp,home}` 路径被标记为危险删除目标）；Windows：`Ctrl+Backspace` 删除上一个单词；Bash 工具的 `env`/`sudo`/`watch`/`ionice`/`setsid` 等包装命令现在也会匹配 deny 规则

## 三、Claude Code v2.1.111 发版【⭐】
**来源：** Claude Code Changelog
**链接：** https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#21111
**关键词：** Claude Code、Changelog、Opus 4.7、xhigh

### 摘要

Claude Code v2.1.111 是一个大版本，主要变化：

**Claude Opus 4.7 xhigh 上线！** 现在可以使用 `/effort` 来调节速度和智能度的权衡。Max 订阅者使用 Opus 4.7 时可以使用 Auto 模式。新增 `xhigh` effort 级别，位于 `high` 和 `max` 之间，通过 `/effort`、`--effort` 或模型选择器调用。

**全新功能：**
- `/ultrareview`：在云端运行综合代码审查，使用并行多 Agent 分析和批判，支持查看当前分支或指定 PR
- `/less-permission-prompts`：扫描 transcript 中的常见只读 Bash 和 MCP 调用，为 `.claude/settings.json` 提出优先级的允许列表
- Windows：PowerShell 工具逐步推出（需要 `pwsh`）
- `/effort` 不带参数时现在打开交互式滑块，用方向键在级别间切换，Enter 确认

**效率改进：** 无需参数的 Bash 命令（如 `ls *.ts`）和 `cd <项目目录> &&` 开头的命令不再触发权限确认。Ctrl+U 现在清空整个输入缓冲区（之前是删除到行首）。
