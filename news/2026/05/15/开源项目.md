## 一、Announcing Genkit Middleware: Intercept, extend, and harden your agentic apps【⭐⭐】
**来源：** Google AI
**链接：** https://developers.googleblog.com/announcing-genkit-middleware-intercept-extend-and-harden-your-agentic-apps/
**关键词：** Genkit、Google、Middleware、AI Agent、TypeScript、Go、Dart、Python
### 摘要
Google 开源框架 Genkit 正式发布中间件（Middleware）系统，为AI Agent开发提供了可组合的插件式基础设施。Genkit 是一个支持 TypeScript、Go、Dart 和 Python 的全栈AI应用框架，其核心是一个"工具循环"（tool loop）：模型输出→工具执行→结果反馈→下一轮调用。

中间件在这三个层级接入这个循环：**Generate层**（每轮工具循环一次，用于上下文注入、消息重写）、**Model层**（每次模型API调用，用于重试、回退、缓存）、**Tool层**（每次工具执行，用于人机确认、沙箱隔离）。

预置的中间件包括：自动重试（指数退避+抖动）、模型回退（主模型超限时自动切到备用模型如Claude）、工具审批（仅允许白名单内的工具调用，触发人工确认）、Skills（扫描SKILL.md目录注入到系统提示）、Filesystem（给模型安全的本地文件系统访问权限）。开发者还可以在20行代码内编写自定义中间件，多个中间件可以叠加组合，且所有中间件配置和执行过程都可在Genkit Developer UI中可视化和调试。

这对使用Genkit构建生产级AI Agent的开发者是非常有价值的更新——特别是"人机确认"和"多模型回退"这两个能力，直接解决了Agent在生产环境可靠性的核心痛点。

## 二、obra/superpowers — Agentic Skills 框架与软件开发方法论【⭐⭐】
**来源：** GitHub Trending Daily
**链接：** https://github.com/obra/superpowers
**关键词：** Agent Skills、CLAUDE.md、软件开发方法论、AI开发框架

### 摘要

obra/superpowers 是一个 Agentic Skills 框架和配套的软件开发方法论，由前 OpenClaw 开发者（obra）维护。项目核心理念是：AI Agent 的能力不是靠提示词堆砌的，而是靠可编排的 Skills 模块。

它的做法是把 Agent 能力拆解为 Skills（类似插件），每个 Skill 是一个独立的指令 + 工具组合，Agent 按需加载。配合 CLAUDE.md 文件作为项目级上下文，不同项目可以有不同的超级能力集。这个思路和 Anthropic 的 Claude Code Skills、OpenClaw 的 Skill 系统如出一辙——说明 Skills 正在成为 AI 开发的标准抽象层。

**适合人群**：正在构建 Agent 基础设施的开发者，或者想给自己的 AI 工作流加上模块化能力的团队。

**链接：** https://github.com/rohitg00/agentmemory

### 摘要

agentmemory 自称是"基于真实世界基准测试的排名第一的 AI 编码 Agent 持久化记忆方案"。解决的核心问题是：AI 编码 Agent 每次启动都是一张白纸，项目历史、用户偏好、之前的决策全部丢失。agentmemory 提供了一种持久化记忆层，让 Agent 能记住项目上下文、回话间状态和用户习惯。

### 摘要

Kronos 是一个面向金融市场的 Foundation Model（基础模型）。它把金融数据（价格、交易量、订单流、财务指标）视为一种"语言"，用大模型的方法去理解和预测。和传统时间序列模型（LSTM、Transformer）不同，Kronos 的定位更接近 GPT 之于自然语言——训练一个通用的金融理解模型，可以用于市场预测、风险分析、策略生成等。

这个方向值得关注的原因：金融领域一直是 AI 应用的高价值场景，但之前的尝试多是特定任务的小模型。Foundation Model 的方式如果跑通，可能改变量化金融的玩法。

**链接：** https://github.com/ruvnet/RuView
**关键词：** WiFi感知、空间智能、生命体征监测、无摄像头

### 摘要

RuView 是一个很酷的技术 demo：利用普通 WiFi 信号来做实时空间定位、生命体征监测和存在检测——不需要任何摄像头。核心原理是 WiFi 信号穿过空间时会受人体影响，通过分析信号变化可以推断人的位置、动作甚至心跳和呼吸。

## 十四、github/spec-kit — GitHub 官方的 Spec-Driven 开发工具包【⭐】
**来源：** GitHub Trending Daily
**链接：** https://github.com/github/spec-kit
**关键词：** Spec-Driven Development、GitHub、规格驱动、软件开发方法论

### 摘要

GitHub 官方推出的 spec-kit，帮助团队践行"规格驱动开发"（Spec-Driven Development）方法论。核心理念是先写规格（spec）再写代码，让 AI 编码 Agent（如 Copilot、Claude Code）基于清晰的规格来生成更一致的代码。

GitHub 推出这个工具包表明：随着 AI 编码 Agent 的普及，"先规格后代码"的开发流程可能会重新变成主流——因为 AI Agent 解读规格生成代码比解读烂需求生成代码靠谱得多。
