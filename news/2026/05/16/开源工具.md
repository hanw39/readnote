## 一、CodeGraph — Claude Code 的预索引代码知识图谱【⭐⭐】

**来源：** GitHub Trending Daily
**链接：** https://github.com/colbymchenry/codegraph
**关键词：** 代码知识图谱、Claude Code、工具调用优化、本地索引

### 摘要

CodeGraph 是一个为 Claude Code 设计的预索引代码知识图谱工具，通过在本地构建代码符号关系、调用图(code call graph)和结构索引，让 Explore agent 直接查询图谱而非逐个扫描文件。内部用 npx 一键初始化，安装后运行 `codegraph init -i` 即可为项目建立索引。

**基准测试数据（6个真实仓库）：** 平均减少 **92% 的 tool calls**，速度提升 **71%**。例如在 VS Code 仓库（4000+文件）上，无 CodeGraph 需要 52 次调用、1分37秒；有 CodeGraph 仅用 3 次调用、17 秒。在 Claude Code 自身仓库（Python+Rust）上，从 40 次/1分8秒降到 3 次/39 秒。

核心价值：对大型代码库，Claude Code 的 Explore agent 通常需要反复 grep、glob、Read 来理解代码结构，消耗大量 token。CodeGraph 的预索引让 agent 一次查询就能拿到上下文，实测带宽可节省约 10 倍。100% 本地运行，无需联网。对于经常用 Claude Code 处理复杂项目的开发者来说，这个工具能显著降低 token 消耗和等待时间。

## 二、Open Generative AI — AI 视频平台的开源替代品【⭐⭐】

**来源：** GitHub Trending Daily
**链接：** https://github.com/Anil-matcha/Open-Generative-AI
**关键词：** AI 生成、视频生成、图像生成、开源、Flux、Midjourney、Sora、Veo

### 摘要

一个开源的 AI 图像/视频生成工作台，集成了 200+ 个模型（包括 Flux、Midjourney、Kling、Sora、Veo 等），无内容过滤、不需要订阅费、MIT 许可、支持自托管。

功能涵盖四个工作室：图片生成、视频生成、口型同步（Lip Sync）、影院模式（Cinema）。提供在线托管版（https://dev.muapi.ai/open-generative-ai）和桌面应用（macOS/Windows/Linux 一键安装）。搭配作者的另一项目 `Generative-Media-Skills`，还可以让 Claude Code、Codex 等 AI 编码 agent 通过终端直接调用这 200+ 模型做端到端的媒体生成流水线。

本质上是一个「AI 视频界的 Stable Diffusion WebUI」式的聚合入口，把各家生成模型统一到一个界面里。对有视频/图像生成需求但不希望被封闭生态绑定的用户来说，这是个实用的自由选择。

## 三、witr — 解答"为什么这个进程在运行？"的系统诊断工具【⭐】

**来源：** GitHub Trending Daily
**链接：** https://github.com/pranshuparmar/witr
**关键词：** 系统诊断、进程分析、TUI、运维工具

### 摘要

witr 是一个用 Go 编写的系统诊断 CLI 工具，专注于解答一个核心问题：**"为什么这个进程在运行？"**。传统工具（ps、top、lsof、systemctl、docker ps）只告诉你 _什么_ 在运行，但你需要手动关联多个输出来推断 _为什么_。

witr 把因果链显式化——它会追踪进程的启动链条，解释这个进程是怎么来的、谁启动的、通过什么链路（supervisor、container、service、shell）最终跑起来的。支持 Linux、macOS、Windows、FreeBSD，提供交互式 TUI 模式和标准 CLI 输出。已打包进 brew、conda、aur、winget、npm 等多个包管理器。

对开发者和运维人员来说，面对一个不认识的进程或端口占用时，witr 能省掉手动排查的那几步重复劳动。
